NAME

womm - Works On My Machine

Installs anything that Ansible supports - Using simple manuscripts.

SYNOPSIS

womm --manuscript yaml-file [--[no-]update] [--[no-]install] [--playbook yaml-file] [--gist gist-id] [--[no-]verbose] [--[no-]keep] [--cache dir]

womm --help

Example

  womm --manuscript marcommachine.yml

Will run the default playbook using the marcommachine.yml manuscript

OPTIONS

The following switches exists and can all be overridden.

--manuscript yaml-file

Where yaml-file is the name of the manuscript to run.

This is a required parameter.

--[no-]update

Determins wether or not to update (git pull) the repo before execution.

Default is --no-update

--[no-]install

Will install womm and the dependencies on your machine - if they are not already installed.

Default is --install

--playbook playbook

Use this setting to run a different ansible playbook than the default.

The playbook will habe to be in the same folder as the (womm) script

Default is ./macascode.yml

--gist gist_id

Run the manuscript in the specified gist

--[no-]verbose

Will run in verbose mode.

Default is --no-verbose

--[no-]keep

Determins wether or not to keep files pulled from gist locally after the execution or not.

Default is --no-keep

--cache dir

Specifies the dir to use for pulling files from gists.

Default is ./gist

--help

Prints the manual page.
